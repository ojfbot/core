Review the given ADR for completeness and quality.

## Required sections

1. **Title** — ADR number + descriptive name
2. **Status** — Proposed, Accepted, Deprecated, or Superseded (with link)
3. **Context** — The problem or situation that prompted this decision. Must include *why* a decision is needed now.
4. **Decision** — What was decided. Clear, unambiguous, actionable.
5. **Consequences** — Both positive and negative. What becomes easier? What becomes harder? What new constraints exist?

## Quality checks

- Context explains the *forces* at play, not just the problem statement
- Decision is specific enough that two engineers would implement it the same way
- Consequences include at least one trade-off or risk
- No implementation details that belong in code comments or docs
- References related ADRs if they exist
- Date is present and accurate

## Output

Rate: **PASS** / **PASS WITH NOTES** / **NEEDS REVISION**

For each issue, state what section needs work and why.
