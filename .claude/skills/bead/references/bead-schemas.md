# Bead schemas

All beads share a common frontmatter shape; the `type` field selects which fields are required and which are optional. Body is markdown. Frontmatter is YAML.

## Common frontmatter

```yaml
---
id: <auto-generated>              # YYYYMMDD-HHMM-<type>-<short-slug>
type: brief | report | decision | discovery
title: "Short, scannable title"
actor: chat-claude | code-claude | <human> | <named-agent>
session_id: <iso-timestamp>       # links beads from the same session
refs:                             # cross-references; each is a typed URI
  - github:owner/repo#42
  - file:src/path/to/file.ts:Lstart-Lend
  - bead:<id>
  - adr:0006
  - url:https://...
hook: <hook-id-or-null>           # the work assignment this bead belongs to
status: live | closed | superseded
created_at: <iso-timestamp>
labels:                           # free-form, optional
  project: cozy-beaver
  phase: 1
  <any other tag>: <value>
---
```

## Type-specific shape

### brief — chat → code (or any → any) handoff

Required additional frontmatter:
- `to: <recipient-actor>` — who's receiving the brief

Required body sections:
- **Context** — what's already true (the receiver doesn't know your conversation history)
- **Goal** — what to do
- **Acceptance criteria** — how we know it's done
- **References** — issues, files, prior beads, ADRs
- **Flag back** — when to surface a question rather than proceed; what *not* to decide unilaterally

Optional body sections:
- **Constraints** — what to avoid; conventions to follow
- **Time-box** — if this is a spike with an explicit exit criterion

### report — code → chat (or any → any) return

Required additional frontmatter:
- `responding_to: <bead-id>` — the brief this report closes (if any). Null if unprompted.

Required body sections:
- **What got done** — concrete, terse, link-heavy
- **What's open** — anything deferred; anything that surprised; anything blocked

Optional body sections:
- **What surprised** — the moments where reality diverged from the brief
- **Decisions made** — links to decision beads, do not duplicate them inline
- **Discoveries** — links to discovery beads, do not duplicate them inline
- **Recommended next session** — what the next chat or code session should focus on

### decision — any session captures an ADR-worthy moment

Required body sections (light ADR shape):
- **Context** — what was being decided and why now
- **Options considered** — at least two
- **Decision** — the call
- **Consequences** — what this commits us to and what it forecloses

Optional frontmatter:
- `supersedes: <bead-id>` — if this overrides a prior decision

Decision beads should typically be promoted to a real ADR in `decisions/adr/` once they're stable. The bead is the *capture at the seam*; the ADR is the *committed artifact*. The bead can stay; the ADR is the canonical reference.

### discovery — any session captures a non-obvious gotcha

Required body sections:
- **The gotcha** — what's surprising, in one or two sentences
- **How to recognize it** — symptoms, error messages, observable signals
- **The fix** — what to do about it

Optional body sections:
- **Where it should land long-term** — CLAUDE.md? README? ADR? Inline comment? Discovery beads are *temporary* by design; they should migrate into project documentation once stable.

## Filename convention

`<YYYYMMDD>-<HHMM>-<type>-<short-slug>.md`

Example: `20260427-1430-brief-execute-phase-1-punch-list.md`

The filename is derived from the `id` field. Do not rename files manually after creation; the filename and the `id` field must stay in sync (the `replay.py` script depends on this).

## Status semantics

- **live** — current and authoritative
- **closed** — work referenced in this bead is complete; the bead remains as history
- **superseded** — a newer bead overrides this one (linked via the new bead's `supersedes` field)

Beads are never deleted. Status transitions are explicit edits to a single field.
