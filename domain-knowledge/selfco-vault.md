# selfco LLM Wiki — architecture brief

`~/selfco` (`${SELFCO_VAULT:-~/selfco}`) is a personal knowledge base built as **Andrej Karpathy's "LLM Wiki"**
(<https://gist.github.com/karpathy/442a6bf555914893e9891c11519de94f>): _"Obsidian is the IDE; the LLM is the
programmer; the wiki is the codebase."_ You curate raw material; the LLM owns and maintains `wiki/`. It parallels
all `ojfbot` work *and* free-form research, and doubles as a navigable graph UI (Obsidian graph view + Excalibrain).
Complementary to `daily-logger` (chronological published blog) and `/bead` (per-repo session handoff). See ADR-0085.

It's its own git repo, not symlinked into ojfbot. **Not** called a "second brain" in code/docs — "the vault" / "the wiki".

## Layers

- **`raw/`** — append-only, immutable source materials (articles, PDFs, clippings, chat/research dumps, notes you
  authored). The single source of truth for everything the wiki was built from. The LLM reads it, never edits/deletes
  it. `raw/assets/` for binaries.
- **`wiki/`** — LLM-owned, compiled, interlinked:
  - `index.md` — master catalog; every page reachable from here; the hub node in the graph.
  - `log.md` — append-only ledger; `## [YYYY-MM-DD] <op> | <title>` (ops: ingest|research|query|lint|sync|session|init); never edited/reordered.
  - `sources/<slug>.md` — one summary page per `raw/` item (and per "virtual source" — a `sync` run, etc.).
  - `entities/<slug>.md` — people, orgs, products, tools — **and every ojfbot repo** (`kind: repo`, carries `repo/ports/status/last_synced`).
  - `concepts/<slug>.md` — ideas, frameworks, theories.
  - `synthesis/<slug>.md` — comparisons, themes, roadmaps (incl. `ojfbot-adrs.md` linking `core/decisions/adr/`).
- **`~/selfco/CLAUDE.md`** — **the schema**: folder roles, page schemas, the workflows, "what this is NOT". Self-contained,
  so any agent (Claude Code, Codex) opening `~/selfco` works without the ojfbot skills. Generated by `init-vault.py`
  from `core/.claude/skills/vault/templates/vault-claude-md.md`.
- **`.obsidian/`** — graph view tuned for the "orbiting clusters" look (colour groups by folder path; `index.md` hub;
  node size by link count); bundled plugins fetched into `.obsidian/plugins/` (gitignored): Excalibrain (+Excalidraw),
  Obsidian Mind Map, Graph Analysis, Persistent Graph. Graphify/TheBrain optionally pointed at the folder externally.
- `templates/` — page scaffolds (`source.md`, `entity.md`, `concept.md`, `synthesis.md`).
- `prompts/` — reusable copy-paste prompts; `session-handoff.md` = the export prompt for foreign chat agents (Dia, claude.ai) → emits a handoff bundle you `ingest`.

There is no `Areas/` or `Sessions/` — that was the v1 layout, migrated into `wiki/` by `migrate-v1.py`.

## The `/vault` skill (`core/.claude/skills/vault/`) — thin wrapper

Auto-symlinked into sibling repos by `install-agents.sh`. Reads `~/selfco/CLAUDE.md` first, then runs the right
script + authors the pages. Modes:

| Mode | What |
|---|---|
| `init` | `init-vault.py`: scaffold `raw/`+`wiki/`, write `~/selfco/CLAUDE.md` (the schema), `.obsidian/` config + templates + README + `.gitignore`, `git init`, fetch Obsidian plugins, seed a `wiki/entities/<repo>.md` stub per `~/ojfbot/*/.git` repo (skips worktrees). Idempotent. |
| `ingest <path\|url>` | Karpathy's core loop: land the source in `raw/` (`ingest.py` helps download/copy + stub the source page) → read & discuss → write `wiki/sources/<slug>.md` → update the entity/concept/synthesis pages it bears on (10–15) → update `wiki/index.md` → append `## [date] ingest | <title>` to `wiki/log.md`. |
| `research <topic>` | `ingest`, but the skill fetches the sources (WebSearch/WebFetch/context7); each source → `raw/` + a `sources/` page; then the `concepts/`/`synthesis/` page. |
| `query <question>` | Answer from `wiki/` with `[[page]]`/`[[sources/…]]` citations; file substantive explorations back as a `synthesis/`/`concepts/` page; log it. |
| `lint [--fix]` | `lint.py` (orphans, broken `[[links]]`, `raw/`-without-`sources/`) + the LLM's semantic checks (contradictions, stale claims, missing cross-refs). Report; mutate only with `--fix`; log it. |
| `sync [--since=7d]` | `collect.py` digest of the ojfbot activity feed → update repo `wiki/entities/*` (Current state, recent work, `status`/`last_synced`) → `wiki/sources/sync-<date>.md` → refresh `index.md` → fold the hook's `## [date] session` stubs → append `## [date] sync | …` to `log.md`. |
| `handoff` | Fold the current session into the wiki — a handoff bundle (summary + the full text of every artifact produced) → `raw/session-<date>-<slug>.md` (source of record) + each artifact → `raw/<slug>.md` + a `wiki/sources/<slug>.md` summary + entity/concept/synthesis updates + a `## [date] ingest \| <session> (handoff)` log entry. = `ingest` where the source is the conversation. (For a *foreign* chat agent: the copy-paste prompt `~/selfco/prompts/session-handoff.md` makes it emit the bundle; bring it back and `ingest` it.) |
| `orient` | Read `wiki/index.md` + recent `wiki/log.md`; surface what the wiki knows. |
| `note <title>` | Quick capture → `raw/<slug>.md` (default) or a `wiki/<type>/` stub. |

Scripts (read/scaffold only; the LLM writes pages): `init-vault.py`, `migrate-v1.py` (one-shot v1→v2),
`collect.py` (read-only activity digest — see `core/.claude/skills/vault/knowledge/wiki-schema.md` for the data
sources), `install-obsidian-plugins.sh`, `lint.py`, `ingest.py`.

## `sync` data sources (read-only)

`~/.claude/projects/*/`, `~/.claude/session-telemetry.jsonl`, `~/.claude/skill-telemetry.jsonl`,
`~/.claude/history.jsonl` (summarized, never quoted), `git -C ~/ojfbot/<repo> log` for each `~/ojfbot/*/` whose
`.git` is a *directory* (skip linked worktrees), `~/ojfbot/<repo>/.handoff/*.md` (link, don't copy),
`~/ojfbot/<repo>/CLAUDE.md` (for entity overviews + ports), `~/ojfbot/core/decisions/adr/[0-9]*.md` (for the
`ojfbot-adrs` synthesis page), and `~/selfco/wiki/log.md` (existing `## [date] session` stub lines).

## Opt-in automation

`core/scripts/hooks/vault-session.sh` — `SessionEnd` hook; appends a `## [date] session | <repo>@<branch>` stub to
`~/selfco/wiki/log.md` (no LLM, no network; no-ops if `wiki/log.md` absent). **Not installed by default** —
`install-agents.sh --user-scope --with-selfco` installs it (and symlinks `/vault` into `~/.claude/skills/` + runs
`init-vault.py`).

## Access from the Claude apps (ADR-0070)

`/vault` (the Claude-Code skill above) runs on the Mac. To tend the vault from the Claude *apps* — claude.ai web, the
iPhone app, the Claude Desktop Mac app — there's a connector + a consumer-app Agent Skill. **Phase A (now):**
`~/selfco` is mirrored to a private GitHub repo `ojfbot/selfco`; Anthropic's official **GitHub connector** (scoped to
that repo) gives the apps read/write access — every write is a commit. **GitHub is the source of truth; `~/selfco` is
a clone** — the Mac-side `/vault` (and `init-vault.py`) `git pull --rebase --autostash` before / `git push` after when
`$V` has a remote; `scripts/autocommit.sh` covers writes that bypass the skill (Mac-local `mcp-obsidian`, hand-edits).
The Mac desktop app also gets a local **`mcp-obsidian`** server (Obsidian "Local REST API" plugin + `MarkusPfundstein/mcp-obsidian`)
for a live-index UX. The **`/vault` Agent Skill** (`core/.claude/skills/vault/consumer/SKILL.md`, uploaded to the
Anthropic account) is connector-agnostic — reads the vault's `CLAUDE.md`, writes via whichever connector is attached;
covers `ingest`/`query`/`note`/`orient` (`init`/`sync`/full web-`research` stay in Claude Code on the Mac). **Phase B
(later):** a locally-hosted obsidian-mcp behind Cloudflare Tunnel + Access added as a custom remote MCP connector — the
Agent Skill is unchanged. Full setup + the migration path: `core/.claude/skills/vault/knowledge/connectors.md`.
