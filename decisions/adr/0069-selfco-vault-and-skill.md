# ADR-0069: The `selfco` LLM Wiki and the `/vault` skill

Date: 2026-05-11 (revised 2026-05-12 ‚Äî adopted Karpathy's LLM-Wiki model before merge)
Status: Accepted
OKR: 2026-Q2 / O-Knowledge / KR-cross-project-recall
Commands affected: /vault (new), /daily-logger (distinction clarified), /bead (composition)
Repos affected: core (skill, hook, install-agents, catalog, domain-knowledge), all sibling repos (skill symlink, selfco-vault.md), new repo ~/selfco

---

## Context

Work across the ojfbot cluster generates signal scattered across `~/.claude/projects/*/`, `session-telemetry.jsonl`,
`skill-telemetry.jsonl`, `history.jsonl`, per-repo `git log`, `.handoff/` beads, and 60+ ADRs. `daily-logger`
already synthesizes a *chronological blog feed* (`log.jim.software`). What was missing is a **lookup-shaped** layer ‚Äî
browsed by *project* and *topic*, covering research as well as shipped code ‚Äî that also doubles as a navigable graph
**UI** (Obsidian's graph view + Excalibrain). The empty `~/selfco` directory is appropriated for it.

A first cut (v1) shipped an activity-shaped vault (per-repo "project notes", dated "session notes", an ADR mirror).
Before merge, the user pointed at **Andrej Karpathy's "LLM Wiki"** pattern
(<https://gist.github.com/karpathy/442a6bf555914893e9891c11519de94f>): _"Obsidian is the IDE; the LLM is the
programmer; the wiki is the codebase."_ ‚Äî an append-only `raw/` source layer, an LLM-owned `wiki/` of compiled
interlinked pages (`sources/`, `entities/`, `concepts/`, `synthesis/`, `index.md`, `log.md`), and a `CLAUDE.md`
*schema* documenting the structure and the **ingest / query / lint** workflows. We adopt that model; the ojfbot
activity stream becomes one input feed into it (every repo is an `entity`, `kind: repo`).

## Decision

`~/selfco` is a standalone git repo organized as a Karpathy LLM Wiki: `raw/` (append-only, immutable; the LLM reads
but never edits) + `wiki/{sources,entities,concepts,synthesis}` + `wiki/index.md` (the hub catalog) + `wiki/log.md`
(append-only `## [YYYY-MM-DD] <op> | <title>` ledger). **The schema lives in the vault** at `~/selfco/CLAUDE.md`
(generated by `init-vault.py` from `core/.claude/skills/vault/templates/vault-claude-md.md`), so any agent ‚Äî Claude
Code or Codex ‚Äî opening `~/selfco` works the Karpathy way without the ojfbot skills installed.

The **`/vault` skill** (`core/.claude/skills/vault/`, auto-symlinked into sibling repos by `install-agents.sh`) is a
*thin wrapper* that runs helper scripts and defers to `~/selfco/CLAUDE.md`. Modes: `init` (scaffold + write the
schema + fetch Obsidian plugins + seed a repo entity stub per `~/ojfbot/*/.git`), `ingest <path|url>` (Karpathy's
core loop: land a source in `raw/` ‚Üí summary page ‚Üí update entity/concept pages ‚Üí index ‚Üí log), `research <topic>`
(ingest but the skill fetches the sources ‚Äî web/context7), `query <question>` (answer from the wiki with citations,
file substantive explorations back), `lint [--fix]` (orphans, broken links, raw-without-source + semantic checks),
`sync [--since=7d]` (fold the ojfbot activity feed into repo entity pages), `handoff` (fold the current session
into the wiki ‚Äî summary + the full text of every artifact produced + a `raw/session-‚Ä¶` source of record + a log
entry; = `ingest` where the source is the conversation), `orient`, `note <title>`. Scripts (`init-vault.py`,
`migrate-v1.py`, `collect.py`, `install-obsidian-plugins.sh`, `lint.py`, `ingest.py`, `autocommit.sh`) only
read/scaffold; the LLM authors all pages. A copy-paste export prompt ‚Äî `~/selfco/prompts/session-handoff.md`
(canonical source `templates/session-handoff-prompt.md`, seeded by `init`) ‚Äî does the same export from a *foreign*
chat agent that has no `/vault` skill (Dia, plain claude.ai): paste it in ‚Üí it emits a handoff bundle ‚Üí bring the
bundle back and `/vault ingest` it (it lands in `raw/` and gets filed).

An **opt-in** `SessionEnd` hook (`core/scripts/hooks/vault-session.sh`, installed only via
`install-agents.sh --user-scope --with-selfco`) appends a `## [date] session | <repo>@<branch>` stub to
`~/selfco/wiki/log.md`, which `/vault sync` later folds in. The graph view (`.obsidian/graph.json`) is tuned for the
"orbiting clusters" look ‚Äî colour groups by folder (`wiki/entities`, `wiki/concepts`, `wiki/sources`,
`wiki/synthesis`, `raw`), node size by link count, `index.md` as the hub; bundled plugins: Excalibrain (+Excalidraw),
Obsidian Mind Map, Graph Analysis, Persistent Graph. Graphify/TheBrain are optional *external* viewers pointed at the
folder ‚Äî no integration code.

## Consequences

### Gains
- One navigable, link-rich, **self-describing** knowledge base ‚Äî works with any agent via the in-vault `CLAUDE.md`,
  not just the ojfbot skill set.
- Karpathy's bookkeeping win: the LLM maintains `index.md`, cross-references, and keeps pages consistent across an
  `ingest` that touches 10‚Äì15 files.
- The ojfbot activity feed (`sync`) and free-form research (`ingest`/`research`) land in the *same* compiled wiki.
- Real graph-UI payoff (folder-keyed colour clusters + `index.md` hub + Excalibrain/Graphify).
- Composes: `/bead` ‚Üí `/vault sync` rolls a handoff into the repo's entity page; `daily-logger` articles can be
  linked from a source page.
- Cheap to adopt: nothing changes for anyone who doesn't run `--with-selfco`; the skill is just available.

### Costs
- Another surface to keep current ‚Äî `ingest`/`sync` must be run for the wiki to reflect reality (mitigated by the
  opt-in hook + suggestion triggers).
- `init` fetches third-party plugin code from GitHub (`.obsidian/plugins/` gitignored, re-fetchable; degrades
  gracefully if offline). Plugins are *downloaded but not auto-enabled* ‚Äî `community-plugins.json` is left empty;
  the user enables what they want via Obsidian's UI, which version-checks. (Auto-enabling a plugin whose
  `minAppVersion` exceeds the installed Obsidian ‚Äî e.g. Excalidraw 2.22.3 needing 1.5.7 on a 1.5.3 install ‚Äî
  crashes the renderer on vault open; the installer now warns about such plugins.)
- Conceptual overlap with `daily-logger` and `/bead`; the in-vault `CLAUDE.md`, the skill prompt, and this ADR draw
  the lines explicitly.
- A v1‚Üív2 migration (`migrate-v1.py`) was needed because v1 shipped the wrong shape first.

### Neutral
- `~/selfco` is its own git repo, not symlinked into ojfbot.
- The vault is deliberately *not* called a "second brain" (user preference); it's "the vault" / "the wiki".
- `selfco-vault.md` is in the universal domain-knowledge symlink set, so every repo sees the architecture brief.

## Alternatives considered

| Alternative | Why rejected |
|-------------|--------------|
| Keep the v1 activity-shaped layout (project/session/decision notes) | Doesn't accumulate free-form research; not the proven Karpathy pattern; user explicitly asked to align with it. |
| Add `raw/`+`wiki/` as a parallel layer alongside v1's `Areas/`/`Sessions/` | Two overlapping structures; the vault is young (~26 notes), so a full restructure is cheap. |
| Keep the canonical schema only in the `/vault` skill, with a pointer `CLAUDE.md` | The vault wouldn't be self-describing for a fresh agent (Codex, or a Claude session without ojfbot skills). Karpathy's model puts the schema *in the vault*. |
| Extend `daily-logger` to also emit wiki pages | Conflates publish-chronological with lookup-compiled; coupling makes both worse. |
| Build a Graphify integration / a hosted web graph UI | Graphify just reads a markdown folder; "keep the markdown clean and point it there" needs no code. A bespoke hosted UI is a separate, larger project (a future `selfco-reader` Frame remote), out of scope here. |
| Vendor (commit) the Obsidian plugins | Bloats the vault repo with third-party JS; gitignored + a re-fetch script is lighter. |
